1.3 Interfata generica Repository

- interfata generica de genul Repository<T, ID>
are toate metodele care trebuie : save, findById, findAll, update, delete

- minim 4 repository
exemple: ClientRepository, SoferRepository, RestaurantRepository, ComandaRepository, ...

- toate interogarile SQL folosesc PreparedStatement
in repository-uri numai asa am facut. toate operatiile crud folosesc preparedstatement
singurul caz in care nu este folosit este la un truc care incarca dinamic schema daca baza
de date nu exista, in care am folosit statement in loc de preparedstatement
pentru ca nu era nici un query string de modificat, doar ceva static

- toate resursele sunt inchise cu try-with-resources
da, am folosit acelasi pattern ca la laborator, cu preparedstatement si resultset

2.  Tranzactii JDBC
- minim o operatie pe mai multe tabele intr-o tranzactie explicita
dap

- set auto commit commit rollback si toate cele
sunt cateva exemple in comandaservice

- exemple concrete din proiect
sunt: checkout, preiaComanda si finalizeazaComanda

- ce fac tranzactiile
checkout face plata, salveaza detaliile comenzii si goleste cosul
preiaComanda actualizeaza comanda si soferul in one go in aceeasi tranzactie
finalizeazaComanda actualizeaza si statusul comenzii (ca a fost livrata) si balanta si disponibilitatea soferului
